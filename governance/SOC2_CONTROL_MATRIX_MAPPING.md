# URAI-Jobs: SOC2 Control Matrix Mapping

This document maps the technical controls and architectural features of the URAI-Jobs platform to the Trust Services Criteria commonly evaluated in a SOC 2 audit. This is not a formal audit but a proactive mapping to demonstrate compliance readiness.

## C - Common Criteria (Security)

| Control ID | Control Category | Risk | URAI-Jobs Control Implementation |
| :--- | :--- | :--- | :--- |
| **CC6.1** | Logical and Physical Access Controls | Unauthorized access to data and system functions. | **Firestore Security Rules**: Enforce strict, default-deny access control based on user authentication, role (`admin`), and tenant (`orgId`).<br>**Firebase Auth**: Manages user identities and authentication.<br>**Multi-Tenant Architecture**: Data is programmatically segregated by `orgId` at the database root, preventing cross-tenant access. |
| **CC6.2** | System Operations | Unauthorized or malicious system activity. | **Cloud Functions**: Backend logic is encapsulated in serverless functions with single responsibilities. Triggers are scoped to specific data paths (`/orgs/{orgId}/...`).<br>**Principle of Least Privilege**: Functions have IAM roles that only grant them the permissions necessary for their operation.<br>**STRIDE Threat Model**: Formal analysis of threats and mitigations is documented. |
| **CC6.3** | System Operations | Unauthorized access to sensitive data (PII). | **Google Cloud Storage Security Rules**: Default-deny policy (`allow read, write: if false;`). Resumes are inaccessible to the public.<br>**Signed URLs**: Short-lived, single-use URLs are generated by secure Cloud Functions for all resume uploads and downloads. This prevents direct, persistent access to PII. |
| **CC7.1** | Change Management | Unauthorized or untested changes introduce vulnerabilities. | **Deterministic Production Builds**: `pnpm install --frozen-lockfile` ensures that all deployments use exact, vetted dependency versions.<br>**Version Control (Git)**: All code changes are tracked. The deployment process is tied to specific commits.<br>**Staging/Emulator Environment**: Firebase Local Emulator Suite is used for local development and testing before deployment. |
| **CC7.2** | Change Management | Changes are not properly authorized. | **Production Release Authority (This Protocol)**: The **MASTER EXECUTION PROTOCOL** serves as a formal process for authorizing and executing production changes.<br>**IAM Roles**: Only authorized personnel have deployment permissions in the Google Cloud project. |

## A - Availability

| Control ID | Control Category | Risk | URAI-Jobs Control Implementation |
| :--- | :--- | :--- | :--- |
| **A1.1** | System Monitoring and Recovery | The platform becomes unavailable to users. | **Serverless Architecture**: Firebase Hosting, Cloud Functions, and Firestore are managed, auto-scaling services provided by Google Cloud, offering high availability by default.<br>**Global CDN**: Firebase Hosting serves static assets via a global CDN, ensuring low-latency access and resilience.<br>**Health Check Endpoint**: An HTTP function (`/health`) is implemented for external monitoring of service status. |

## C - Confidentiality

| Control ID | Control Category | Risk | Confidential information (PII) is disclosed to unauthorized parties. | URAI-Jobs Control Implementation |
| :--- | :--- | :--- | :--- | :--- |
| **C1.1** | Identification and Protection of Confidential Information | Applicant resumes and personal data are exposed. | **Data Classification**: Applicant resumes are explicitly classified as PII and handled through a separate, more secure flow than other data.<br>**Multi-Tenant Isolation**: The `orgId`-based architecture ensures that one tenant's confidential applicant data is not visible to another.<br>**`Applicant Data Flow Diagram`**: This document explicitly details the secure handling of PII from submission to retrieval. |
| **C1.2** | Protection of Confidential Information | Confidential data is accessed in transit or at rest. | **Encryption in Transit**: All traffic is enforced over HTTPS (SSL).<br>**Encryption at Rest**: All data in Cloud Firestore and Google Cloud Storage is encrypted at rest by default by Google Cloud.<br>**Secure Storage**: Resumes are stored in a non-public Cloud Storage bucket with security rules that prevent any direct access. |
