
# URAI-JOBS: A Production-Grade Job Execution System

This repository contains the implementation of URAI-JOBS, a robust and scalable job execution system built on Firebase, Cloud Run, and Firestore. It is designed for asynchronous, long-running tasks and features a resilient, lease-based execution model.

## Architecture

The system follows a distributed architecture centered around Firestore for state management and Cloud Functions for orchestration.

1.  **Client**: Initiates a job by calling the `createJob` callable Cloud Function with a specific job type and payload.

2.  **Firebase Functions**: The core orchestration layer. It validates the job request, creates a job document in Firestore, and places the job in a queue. A separate set of functions manages the queue, leasing jobs to workers, and handling timeouts and failures.

3.  **Cloud Run Workers**: Headless services that perform the actual work. They receive jobs from the function-based orchestrator, execute the specific task (e.g., text-to-speech conversion), and report back the result.

4.  **Firestore**: Acts as the single source of truth for the entire system, storing job state, queue status, results, logs, and user permissions.

5.  **Google Cloud Storage**: Used for storing job artifacts, such as the audio files generated by the `narrator.tts` job.

### Job Lifecycle

1.  **Creation (`PENDING`)**: A job document is created in the `jobs` collection and a corresponding entry is added to the `jobQueue`.
2.  **Leasing (`LEASED`)**: The `processQueueTick` function leases an available job to a worker class, preventing other workers from picking it up.
3.  **Execution (`RUNNING`)**: The `executeJob` function calls the appropriate worker. The worker performs the task.
4.  **Completion (`SUCCESS` or `FAILED`)**: Upon completion, the worker returns a result. A `jobResults` document is created, and the job's status is updated to a terminal state (`SUCCESS`, `FAILED`, `DEAD`).
5.  **Cleanup**: A periodic function (`cleanupTerminalJobs`) removes completed jobs from the queue.

## Core Firebase Functions

The `functions/src` directory contains the core logic for the job system:

*   `createJob`: A callable function that serves as the main entry point for creating new jobs.
*   `processQueueTick`: A scheduled function that runs periodically to find and lease new jobs from the queue.
*   `executeJob`: A function that proxies the execution request to the appropriate Cloud Run worker based on the job type.
*   `handleJobFailure`: Centralized error handler for all job failures.
*   `retryExpiredLeases`: A scheduled function to find jobs with expired leases and requeue them.
*   `cleanupTerminalJobs`: A scheduled function that cleans up jobs that have reached a terminal state from the queue.
*   `systemReconcile`: A scheduled function for system-wide health checks and reconciliation.
*   `getJobStatus`, `cancelJob`: Callable functions for clients to interact with existing jobs.

## First Real Job Type: `narrator.tts`

The first fully implemented job type is `narrator.tts`, which converts text into speech using Google Cloud Text-to-Speech.

**Payload Contract:**

```typescript
interface NarratorTtsPayload {
  text: string;
  voice: string;      // e.g., 'en-US-Wavenet-D'
  locale: string;     // e.g., 'en-US'
  format: 'MP3' | 'OGG_OPUS';
  outputPrefix: string; // GCS prefix to save the file
}
```

## Setup and Deployment

### Prerequisites

*   Node.js (v18 or later)
*   Firebase CLI
*   Google Cloud SDK

### Environment Variables

The following environment variables must be configured.

**For Firebase Functions (`functions/.env`):**

```
NARRATOR_WORKER_URL=https://your-narrator-worker-url.a.run.app
```

**For the Cloud Run Worker (`workers/narrator-worker/.env` or deployment config):**

```
# GOOGLE_APPLICATION_CREDENTIALS=/path/to/your/service-account-key.json (if running locally)
GCS_BUCKET_NAME=your-gcs-bucket-name
```

### Deployment Sequence

1.  **Deploy the Narrator Worker to Cloud Run:**

    ```bash
    cd workers/narrator-worker
    gcloud builds submit --tag gcr.io/[GCP_PROJECT_ID]/narrator-worker
    gcloud run deploy narrator-worker --image gcr.io/[GCP_PROJECT_ID]/narrator-worker --platform managed --region [GCP_REGION] --allow-unauthenticated
    ```
    *Note: Take the URL from the deployment output and add it to the `functions/.env` file.*

2.  **Deploy Firebase Resources:**

    ```bash
    firebase deploy --only functions,firestore
    ```

## Submitting a Job

Jobs are submitted by calling the `createJob` Firebase Callable Function.

```javascript
const createJob = firebase.functions().httpsCallable('createJob');

const jobRequest = {
  type: 'narrator.tts',
  payload: {
    text: 'Hello, world! This is a real job.',
    voice: 'en-US-Wavenet-D',
    locale: 'en-US',
    format: 'MP3',
    outputPrefix: 'tts-outputs/my-first-job'
  }
};

createJob(jobRequest)
  .then(result => {
    console.log('Job created with ID:', result.data.jobId);
  })
  .catch(error => {
    console.error('Error creating job:', error);
  });
```

## Verifying Results

1.  **Check the Job Document**: The job's status will transition `PENDING` -> `RUNNING` -> `SUCCESS` in the `jobs` collection.
2.  **Inspect the Result Document**: A new document will be created in the `jobResults` collection with the ID of the completed job. This contains the result manifest.
3.  **Verify the Artifact**: The generated audio file will be in your GCS bucket at the path specified in the result manifest.
