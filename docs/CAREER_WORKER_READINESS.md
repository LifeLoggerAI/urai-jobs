# Career Worker Readiness

Status: scaffolded
Date: 2026-06-04
Related decision: `docs/product-decisions/PDR-001-autonomous-urai-jobs-v1-v5.md`

## What exists

The repository now contains a dedicated `workers/career-worker` package with:

- TypeScript build configuration.
- HTTP worker entrypoint.
- Health endpoint.
- Job execution endpoint.
- Safe scaffold handlers for the approved career runtime job types.
- Dockerfile for Cloud Run packaging.

## Registered runtime job types

The runtime registry includes these career job contracts:

- `career.profile.summarize`
- `career.fit.score`
- `career.document.parse`
- `career.document.tailor`
- `career.packet.generate`
- `career.followup.plan`
- `career.interview.prep`
- `career.offer.compare`
- `career.spatial.portal.generate`
- `career.passport.export`

## V1 readiness

Before V1 can be called complete:

- [ ] Career worker builds locally.
- [ ] Career worker Docker image builds locally.
- [ ] Career worker is deployed to Cloud Run.
- [ ] `CAREER_WORKER_URL` is configured in production env.
- [ ] Dispatcher routes `career.*` jobs to `CAREER_WORKER_URL`.
- [ ] Production worker health check passes.
- [ ] A `career.profile.summarize` job reaches terminal success.
- [ ] A `career.fit.score` job reaches terminal success.
- [ ] Runtime logs show the career job correlation ID.
- [ ] Release evidence includes worker URL, health response, job ID, logs, and rollback path.

## Current limitation

The career worker package exists, but the dispatcher still needs a safe routing patch so `career.*` jobs use `CAREER_WORKER_URL` instead of falling through to the default worker path.
